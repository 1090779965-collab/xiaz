# 📧 email-automation - 邮件自动处理

## 1. 技能概述

**核心能力**：自动收件、筛选、回复、跟进邮件

**适用场景**：
- 自动回复常见问题邮件
- 定时发送批量邮件
- 跟进未回复的重要邮件
- 邮件分类归档
- 自动提取任务信息

**对比传统方式**：
- 手动处理50封邮件：30分钟
- OpenClaw自动处理：10秒

---

## 2. 核心原理

### 2.1 邮件协议

```
SMTP (发送)  ←→  IMAP/POP3 (接收)
   ↑                    ↑
   └──── 邮件服务器 ────┘
```

### 2.2 配置方式

**方式1：SMTP直连**
```
绑定邮箱：xxx@company.com
SMTP服务器：smtp.company.com
端口：587 (TLS) 或 465 (SSL)
用户名：xxx@company.com
授权码：xxxxxx
```

**方式2：邮件API**
- Gmail API
- Microsoft Graph API
- 飞书邮件API

### 2.3 执行流程

```
获取邮件 → 分析内容 → 判断意图 → 生成回复 → 发送/归档
```

## 3. 实际指令示例

### 示例1：总结未读邮件
```
总结今日未读邮件，标记紧急事项
```

**执行逻辑**：
1. 连接邮件服务器获取未读邮件
2. 提取发件人、主题、摘要
3. 识别紧急邮件（关键词：紧急、截止、Urgent）
4. 生成汇总报告

**输出示例**：
```

🔥 紧急 (2封)：
• 项目截止提醒 - 张三 (截止今天17:00)
• 系统故障 - IT部门 (需要立即处理)

📌 待处理 (5封)：
• 周报汇总 - 李四
• 会议邀请 - 王五
...

📢  newsletters (5封)：
• 技术周报、AI前沿、行业动态
```

### 示例3：跟进邮件
```

```

**执行逻辑**：
1. 搜索已发送邮件

请问贵方对提案有何疑问？如需进一步沟通，我方可安排线上会议。

期待您的回复！
```

## 5. 最佳实践

### ✅ 安全
- 使用应用专用密码（不要用邮箱登录密码）
- 定期检查授权应用
- 敏感操作需要确认

### ✅ 效率
- 设置过滤规则，减少垃圾邮件
- 利用标签/文件夹管理
- 重要邮件设置提醒

### ⚠️ 注意
- 遵守邮件服务商使用规范
- 避免频繁发送被识别为 spam
- 重要邮件建议人工审核

## 6. 常见问题

**Q：企业邮箱无法绑定？**
A：检查是否开启了SMTP/IMAP，可能需要管理员授权

**Q：发送被识别为垃圾邮件？**
A：避免频繁发送、使用真实域名、添加退订链接

**Q：如何取消自动回复？**
A：发送"停止自动回复"或设置生效时间段
