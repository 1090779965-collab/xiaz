# 行业研究

## 核心方法论

### 研究流程

```
1. 定义问题 → 2. 信息收集 → 3. 分类整理 → 4. 结构化输出
```

### 信息源优先级

| 优先级 | 来源 | 可靠性 | 获取方式 |
|--------|------|--------|----------|
| ⭐⭐⭐ | 政府官网/官方政策 | 最高 | 搜索+官网 |
| ⭐⭐⭐ | 行业报告/白皮书 | 高 | 搜索+抓取 |
| ⭐⭐ | 权威媒体/行业媒体 | 中高 | 搜索 |
| ⭐⭐ | 企业官网/财报 | 中 | 搜索+抓取 |
| ⭐ | 社区/论坛/社交媒体 | 中低 | 搜索 |

## 搜索技巧

### 1. 关键词组合

```
# 基础搜索
行业 + 2025/2026
行业 + 趋势/现状/报告
行业 + 政策/法规/标准

# 深度搜索
行业 + 问题/痛点/挑战
行业 + 解决方案/案例
行业 + 技术/方案/实现
```

### 2. 多角度搜索

| 维度 | 搜索词 |
|------|--------|
| 政策 | "化妆品电子标签 政策" |
| 技术 | "二维码 多码合一 技术方案" |
| 案例 | "品牌 防伪码 公众号 案例" |
| 供应商 | "二维码生成服务商 化妆品" |
| 竞品 | "竞品 扫码营销 方案" |

### 3. 交叉验证

重要信息需要**多源验证**：
- 至少2-3个独立信源
- 官方与非官方相互印证
- 时间线验证（最新vs过时）

## 信息收集

### 搜索工具使用

```bash
# 基础搜索
miaoda-studio-cli search-summary --query "关键词" --instruction "总结关键信息"

# 带条件的搜索
miaoda-studio-cli search-summary --query "行业 2025" --instruction "列出主要趋势和关键数据"
```

### 网页抓取使用

```bash
# 抓取具体页面
miaoda-studio-cli web-crawl --url "https://example.com/report" --instruction "提取核心观点和数据"
```

### 搜索决策

| 场景 | 工具选择 |
|------|----------|
| 不知道具体URL，需要搜索 | search-summary |
| 有明确URL，需要详细内容 | web-crawl |
| 多个URL，需要批量处理 | 循环web-crawl |

## 分类整理

### 信息分类框架

```
## 行业概况
- 市场规模
- 增长趋势
- 竞争格局

## 政策监管
- 相关政策
- 合规要求
- 试点/执行

## 技术方案
- 主流技术
- 供应商
- 成本

## 案例参考
- 成功案例
- 失败教训

## 趋势预测
- 短期（1年内）
- 中期（1-3年）
- 长期（3年+）
```

### 整理原则

1. **来源标注**：每条信息标注来源
2. **时间标注**：标注信息时间（避免过时信息）
3. **置信度**：区分事实 vs 分析 vs 预测
4. **结构化**：用表格、列表、框架

## 结构化输出

### 输出模板

```markdown
# [行业/主题]研究报告

## 一、行业概况
[核心数据]

## 二、政策环境
[政策要点]

## 三、技术方案
[技术对比]

## 四、趋势预测
[未来展望]

## 五、建议
[可行动的建议]
```

### 快速总结模板

当信息量大时，用以下结构快速输出：

```
## 核心发现
- 发现1
- 发现2

## 关键数据
- 数据1
- 数据2

## 下一步
- 行动1
- 行动2
```

## 常见行业研究主题

### 1. 政策合规研究
- 搜索官方政策文件
- 理解适用范围和时间线
- 识别企业合规路径

### 2. 技术方案研究
- 主流技术对比
- 供应商调研
- 成本效益分析

### 3. 竞品研究
- 产品/服务分析
- 营销策略研究
- 用户评价收集

### 4. 市场研究
- 市场规模测算
- 增长驱动因素
- 进入壁垒分析

## 注意事项

1. **信息时效性**：优先搜索最近的信息（2024-2026）
2. **来源可靠性**：官方>权威机构>普通媒体
3. **避免偏见**：多角度搜索，避免单一来源
4. **行动导向**：研究目的是指导行动，不是为了研究而研究
