# 任务分解执行

## 核心思想

借鉴CoT（思维链）和ReACT（推理-行动循环）思想：
- **CoT**：将复杂问题拆解为线性推理步骤
- **ReACT**：推理→行动→观察→结论的循环

## 执行流程

### 1. 任务理解（Think）

遇到复杂任务，先理解：
- 用户的**最终目标**是什么？
- 任务的**边界**在哪里？
- 需要什么**工具/资源**？
- 有什么**约束条件**？

### 2. 任务分解（Decompose）

将任务拆解为子任务，格式：

```
## 任务计划

### 子任务1: [具体行动]
- 目标：[该步骤要达成什么]
- 依赖：[需要前置什么]
- 工具：[需要什么资源]

### 子任务2: [具体行动]
...
```

**分解原则**：
- 每个子任务应该是**原子性**的（不可再分）
- 明确**先做什么、后做什么**
- 标注需要**外部工具**的步骤
- 考虑**并行**可能性（独立任务可同时执行）

### 3. 逐步执行（Execute）

按顺序执行子任务，每步：

1. **确认**：明确当前步骤的目标
2. **执行**：调用工具/生成内容
3. **观察**：检查结果是否符合预期
4. **决策**：继续下一步 / 调整计划 / 求助用户

### 4. 检查与整合（Review）

完成所有子任务后：
- 最终结果是否达到目标？
- 中间有遗漏吗？
- 各步骤衔接是否流畅？
- 需要补充什么？

## 分解策略

### 策略1：流程分解
适用于有明确流程的任务（如写报告、办手续）

```
1. 收集信息
2. 整理分析
3. 撰写草稿
4. 审核修改
5. 完成输出
```

### 策略2：要素分解
适用于需要从多个维度处理的任务（如分析一个方案）

```
1. 目标维度：想要达成什么
2. 成本维度：需要投入什么
3. 风险维度：可能有什么问题
4. 收益维度：能获得什么
```

### 策略3：递归分解
适用于不确定边界的复杂任务，逐步细化

```
大任务
└─ 子任务A（仍复杂）
   └─ 子任务A1（可执行）
   └─ 子任务A2（可执行）
└─ 子任务B（可执行）
```

### 策略4：多Agent分解（2025进阶）

当任务需要不同能力时，可考虑：
- **规划Agent**：负责拆解任务
- **执行Agent**：负责具体执行
- **检查Agent**：负责审核结果

## 动态调整

分解不是一次性完成的，可能需要动态调整：

| 情况 | 应对 |
|------|------|
| 执行中发现遗漏 | 插入新子任务 |
| 步骤太多太杂 | 合并相关步骤 |
| 方向偏离目标 | 回退并重新规划 |
| 用户补充信息 | 更新计划 |

## 使用示例

### 示例1：用户说"帮我写一个爬虫"

**思考**：这是一个开发任务，需要明确需求

**分解**：
```
## 任务计划

### 1. 确认需求
- 目标：明确要抓取什么网站、什么数据
- 对话：询问目标URL、数据字段、保存格式

### 2. 分析网页
- 目标：了解网页结构
- 行动：获取网页HTML，分析DOM结构
- 工具：浏览器/curl

### 3. 编写代码
- 目标：生成可运行的爬虫脚本
- 依赖：已完成需求确认和网页分析
- 工具：Python + requests/selenium

### 4. 测试运行
- 目标：验证爬虫正常工作
- 依赖：代码已完成
- 行动：运行脚本，检查输出

### 5. 数据处理
- 目标：清洗和格式化数据
- 依赖：数据已抓取
- 输出：CSV/JSON/数据库
```

### 示例2：用户说"帮我分析一下这个方案"

**分解**：
```
## 任务计划

### 1. 理解方案
- 目标：明确方案内容、背景、目标
- 行动：阅读方案文档

### 2. 要素分析
- 目标：从多维度评估方案
- 维度：
  - 可行性（技术、资源、时间）
  - 风险点（潜在问题）
  - 收益（预期效果）
  - 成本（投入）

### 3. 对比分析
- 目标：客观评估方案优劣
- 行动：列出优点和不足

### 4. 建议输出
- 目标：给出结论和建议
- 格式：总结 + 可行动的具体建议
```

## 常见陷阱

| 陷阱 | 避免方法 |
|------|----------|
| 过度分解 | 保持子任务"可执行"粒度，不要细化到琐碎步骤 |
| 忽视依赖 | 明确标注任务间的依赖关系 |
| 一次性写完 | 每步都要有反馈，不要闷头干到底 |
| 不检查结果 | 每步都确认是否符合预期 |

## 检查清单

执行前检查：
- [ ] 目标明确吗？
- [ ] 分解完整吗？
- [ ] 依赖关系理清了吗？

执行中检查：
- [ ] 当前步骤完成了吗？
- [ ] 结果符合预期吗？
- [ ] 需要调整计划吗？

执行后检查：
- [ ] 达到最终目标了吗？
- [ ] 用户满意吗？
- [ ] 有需要补充的吗？
