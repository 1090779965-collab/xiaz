## 2. 核心原理

### 2.1 文件扫描
```javascript
// 伪代码示例
const fs = require('fs');
const path = require('path');

async function scanDirectory(dirPath) {
  const files = [];
  const items = fs.readdirSync(dirPath, { withFileTypes: true });

  for (const item of items) {
    const fullPath = path.join(dirPath, item.name);
    if (item.isDirectory()) {
      files.push(...await scanDirectory(fullPath));
    } else {
      files.push({
        name: item.name,
        path: fullPath,
        size: fs.statSync(fullPath).size,
        ext: path.extname(item.name).toLowerCase(),
        modified: fs.statSync(fullPath).mtime
      });
    }
  }
  return files;
}
```

### 2.2 文件分类策略

| 分类维度 | 规则示例 | 目标文件夹 |
|----------|----------|------------|
| 文件类型 | .pdf, .docx, .jpg | /Documents/PDFs, /Images |
| 日期 | 修改时间在7天内 | /Recent/2025-03-24 |
| 大小 | >10MB 的大文件 | /LargeFiles |
| 关键词 | 文件名含"发票" | /Finance/Invoices |
| 来源 | 来自下载文件夹 | /Downloads/Organized |

### 2.3 执行流程

```
扫描 → 分析 → 决策 → 执行 → 报告
```

## 3. 实际指令示例

### 示例1：整理下载文件夹
```
整理我的下载文件夹，按文件类型分类到桌面"整理后"文件夹
```

**执行逻辑**：
1. 扫描 `~/Downloads`
2. 识别扩展名（.pdf, .jpg, .mp4, .zip等）
3. 创建对应分类文件夹
4. 移动文件（保留原文件结构）
5. 生成整理报告

**输出示例**：
```
✅ 整理完成！
📊 扫描文件：247个
📁 创建分类：8个（文档、图片、视频、音频、压缩包、程序、其他）

⚠️ 跳过：12个（正在使用/权限不足）
📄 报告已保存：~/Downloads/整理报告_2025-03-24.md
```

### 示例3：批量重命名
```
把下载文件夹里所有 Screenshot 截图重命名为 "截图_日期_序号"
```

**执行逻辑**：
1. 筛选文件名匹配 `Screenshot*.png`
2. 按修改时间排序
3. 生成新名称：`截图_20250324_001.png`
4. 批量重命名（支持预览模式）

**输出示例**：
```
🔄 批量重命名完成：
Screenshot 2025-03-24 at 10.30.00.png → 截图_20250324_001.png
Screenshot 2025-03-24 at 11.15.00.png → 截图_20250324_002.png
共处理 5 个文件
```

---

### 示例4：清理重复文件
```
检查下载文件夹，找出重复文件并列出（不删除）
```

**执行逻辑**：
1. 计算每个文件的MD5哈希值
2. 相同哈希值的文件判定为重复
3. 列出重复组（保留最早版本）

**输出示例**：
```
🔍 发现 2 组重复文件：

组1：setup_v1.0.exe (15.2 MB)
  - ~/Downloads/setup_v1.0.exe
  - ~/Desktop/backup/setup_v1.0.exe

组2：report.pdf (2.1 MB)
  - ~/Downloads/report.pdf
  - ~/Documents/report_copy.pdf

💡 建议：删除重复版本，保留最早的文件
```
