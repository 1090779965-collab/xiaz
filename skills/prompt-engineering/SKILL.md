# Prompt工程

## 核心原则

### 1. 清晰明确

| ❌ 模糊 | ✅ 清晰 |
|---------|---------|
| 写点东西 | 写一首关于春天的诗，4句，每句5个字 |
| 帮我整理 | 把以下数据整理成表格，包含姓名、年龄、部门三列 |
| 解释一下 | 用通俗易懂的语言解释什么是AI，不超过100字 |

### 2. 结构化输出

```
任务 + 格式 + 约束

示例：
- "列出3个优点，用 bullet point 格式"
- "用JSON格式输出，包含name和age字段"
- "分两部分：先解释概念，再给出例子"
```

### 3. 给出示例

**Few-Shot Prompting**：

```
任务：把中文翻译成英文

示例：
- 你好 → Hello
- 谢谢 → Thank you

现在请翻译：早上好
```

## Prompt结构

### 基础结构

```
[角色] + [任务] + [格式] + [约束] + [示例]

示例：
[角色]你是一个专业的营销文案专家
[任务]为产品写一段广告文案
[格式]不超过50字
[约束]突出性价比
[示例]参考：XXX品牌，"每天只需1元"
```

### 进阶结构

```
# 角色
你是一位资深的产品经理

# 背景
我们的产品是面向年轻人的智能手表

# 任务
请写3条朋友圈广告文案

# 要求
- 突出时尚、科技感
- 适合年轻人
- 每条不超过20字

# 输出格式
1. xxx
2. xxx
3. xxx
```

## 常用技巧

### 1. 思维链（CoT）

```
问题：小明有5个苹果，小红给了他3个，小明吃了2个，还剩多少？

计算步骤：
1. 原有5个
2. +3个 = 8个
3. -2个 = 6个
答：还剩6个苹果
```

### 2. 角色扮演

```
你是一位有10年经验的资深律师，精通合同法和劳动法。
请以专业角度分析以下案例...
```

### 3. 分解任务

```
请完成以下任务：
1. 第一步：xxx
2. 第二步：xxx
3. 第三步：xxx
```

### 4. 指定风格

| 风格 | 示例 |
|------|------|
| 专业正式 | "使用商务用语，正式书面风格" |
| 轻松活泼 | "用年轻人语气，网络流行语" |
| 学术严谨 | "引用数据，保持客观中立" |
| 讲故事 | "用故事形式，生动有趣" |

### 5. 约束条件

```
- 字数限制：不超过200字
- 格式要求：JSON/XML/表格
- 内容约束：不提及价格
- 语言：中文输出
```

## 场景模板

### 1. 问答助手

```
你是一位[领域]专家。

请回答用户问题：
{用户问题}

要求：
- 回答准确专业
- 如有不确定，注明"不确定"
- 保持友好语气
```

### 2. 内容生成

```
请生成[类型]内容：

主题：[具体主题]

要求：
- [要求1]
- [要求2]
- 长度：[字数]

风格：[指定风格]
```

### 3. 数据处理

```
请处理以下数据：

原始数据：
{data}

任务：
- [任务1]
- [任务2]

输出格式：
{格式要求}
```

### 4. 代码生成

```
请用[语言]写一个[功能]程序：

需求：
1. [需求1]
2. [需求2]

要求：
- 简洁易读
- 包含注释
- 错误处理
```

## 优化技巧

### 迭代优化

| 步骤 | 操作 |
|------|------|
| 1 | 写第一版Prompt |
| 2 | 测试输出 |
| 3 | 分析问题 |
| 4 | 针对性优化 |
| 5 | 再次测试 |

### 问题诊断

| 问题 | 解决 |
|------|------|
| 输出太长 | 加字数限制 |
| 输出太短 | 要求展开说明 |
| 格式不对 | 明确格式要求 |
| 理解有误 | 增加示例 |
| 不够专业 | 指定角色/背景 |

### 检查清单

- [ ] 任务是否清晰？
- [ ] 格式是否明确？
- [ ] 约束是否完整？
- [ ] 示例是否有效？
- [ ] 角色是否合适？

## 高级技巧

### 1. 系统Prompt + 用户Prompt

```
系统：你是一位助手，擅长用简洁的语言解释复杂概念
用户：什么是量子计算？
```

### 2. 思维树（ToT）

```
问题：如何解决城市交通拥堵？

方案1：增加公共交通
- 优点：减少私家车
- 缺点：需要大量投资

方案2：限制私家车出行
- 优点：效果明显
- 缺点：影响出行

方案3：智能交通系统
- 优点：提高效率
- 缺点：技术要求高

请综合分析给出建议
```

### 3. ReACT模式

```
问题：今天天气怎么样？

思考：我需要查询天气信息
行动：调用天气API
观察：返回结果是晴天，25度
回答：今天是晴天，气温25度，适合外出
```

## 常见错误

| 错误 | 修正 |
|------|------|
| 过于笼统 | 具体化要求 |
| 缺少示例 | 增加Few-Shot |
| 约束太多 | 抓住核心要求 |
| 角色冲突 | 明确单一角色 |
| 格式混乱 | 结构化输出 |
